# 勤怠管理アプリ — 要求仕様 Q&A

## 対象ユーザー

[Question] このアプリの利用者は誰ですか？（例: 中小企業の社員＋管理者、特定部署のみ、など）
[Answer]大企業の社員全員＋管理者

[Question] 想定ユーザー数はどの程度ですか？（例: 10人、100人、1000人以上）
[Answer]1,000人

## 基本機能

[Question] 最低限必要な機能はどれですか？（複数選択可）
- 出勤打刻 / 退勤打刻
- 休憩開始 / 休憩終了の打刻
- 勤務時間の自動計算
- 月次の勤怠一覧表示
- CSV エクスポート
- 有給休暇の申請・管理
- シフト管理
- 残業申請・承認ワークフロー
[Answer]
- 出勤打刻 / 退勤打刻
- 休憩開始 / 休憩終了の打刻
- 勤務時間の自動計算
- 月次の勤怠一覧表示
- 有給休暇の申請・管理
- 残業申請・承認ワークフロー
- 承認者用の画面
- 管理者用の画面

## 打刻

[Question] 打刻方法はどうしますか？（例: Web画面のボタン押下のみ、GPS付き、ICカード連携）
[Answer]ICカード連携

[Question] 打刻の修正（打刻忘れ等）は誰が行えますか？（例: 本人が申請→管理者承認、管理者のみ直接修正）
[Answer]本人が申請→管理者承認

## 勤務ルール

[Question] 所定労働時間は何時間ですか？（例: 8時間/日、7.5時間/日）
[Answer]7.5時間/日

[Question] フレックスタイム制はありますか？コアタイムは？
[Answer]あり、10:00-15:00

[Question] 残業の計算ルールは？（例: 所定超過分を自動計算、申請ベースのみ）
[Answer]所定超過分を自動計算

## 権限・ロール

[Question] ロール（権限の種類）はどう分けますか？（例: 一般社員 / 管理者 の2種、部門長を含む3種）
[Answer]一般社員 / 課長代行以上の役職者(承認者) / 管理者 の3種

[Question] 管理者は全社員の勤怠を見られますか？それとも自部門のみ？
[Answer]管理者は全て、承認者は自部門のみ

## 認証

[Question] ログイン方式は？（例: メールアドレス+パスワード、社員番号+パスワード、SSO連携）
[Answer]SSO連携

## 技術スタック

[Question] 以下の構成で問題ないですか？
- Backend: Java / Spring Boot
- Frontend: TypeScript / Next.js
- DB: PostgreSQL
- 認証: Spring Security（セッションベース）

[Question] 他に考慮すべき要件はありますか？（例: モバイル対応必須、既存システムとの連携、法令対応）
