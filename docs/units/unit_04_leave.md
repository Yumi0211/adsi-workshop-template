# Unit 04: 有給休暇ドメイン

## 概要

有給の自動付与・残高管理・申請機能。

## ユーザーストーリー

- 社員として、有給の残日数を確認したい
- 社員として、有給（全休/半休）を申請したい
- 社員として、年5日義務の達成状況を確認したい

## テーブル

- leave_balances

## API

| Method | Path | 説明 |
|--------|------|------|
| GET | /api/v1/leaves/balance | 自分の有給残高 |
| POST | /api/v1/leaves/requests | 有給申請 |
| GET | /api/v1/leaves/requests | 自分の有給申請一覧 |

## Entity

- LeaveBalance

## Service

- LeaveService（付与・残高計算・消化・時効管理）

## ドメインルール

- 入社日起算で勤続年数に応じた法定日数を自動付与
- 繰越: 前年度の未消化分は翌年度に繰越（2年で時効）
- 全休 = 1.0日消化、半休 = 0.5日消化
- 残日数が足りない場合は申請不可
- 年度内に5日未取得の場合アラート対象（アラート生成は unit_06）

## 依存

- unit_01_employee（Employee FK、入社日参照）

## 完了条件

- [ ] 有給残高が正しく取得できる
- [ ] 有給申請が作成できる（approval_requests への登録は unit_05 と連携）
- [ ] 承認時に残日数が減算される
- [ ] 残日数不足で申請がエラーになる
- [ ] 繰越・時効の計算が正しい
