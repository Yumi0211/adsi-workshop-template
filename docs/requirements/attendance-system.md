# 勤怠管理システム — 要求仕様書

## 概要

大企業（約1,000名）向けの勤怠管理Webアプリケーション。
ICカード打刻とWeb打刻を併用し、フレックスタイム制・法令準拠の勤怠管理を実現する。

## 技術スタック

| レイヤー | 技術 |
|---------|------|
| Backend | Java / Spring Boot |
| Frontend | TypeScript / Next.js（レスポンシブWeb） |
| DB | PostgreSQL |
| 認証 | Microsoft Entra ID (Azure AD) SSO + Spring Security |

## ロールと権限

| ロール | 説明 | 権限 |
|--------|------|------|
| 一般社員 | 全社員 | 自分の打刻・勤怠確認・有給申請・打刻修正申請 |
| 承認者 | 課長代行以上の役職者 | 自部門の勤怠閲覧・各種申請の承認 |
| 管理者 | 人事・総務部門 | 全社員の勤怠閲覧・マスタ管理・レポート・アラート確認 |

## 組織構造

- 4階層: 会社 → 本部 → 部 → 課
- 兼務あり（1人が複数部門に所属可能）
- 承認者は自部門（配下）のみ閲覧・承認可能

## 機能要件

### F-01: 打刻

#### F-01-1: ICカード打刻

- 専用ICカードによる出勤・退勤打刻
- カードリーダーは入口に1台設置
- 打刻時刻をサーバーに記録する

#### F-01-2: Web打刻（フォールバック）

- Web画面のボタンによる出勤・退勤打刻
- ICカード忘れ時や在宅勤務時に使用
- 打刻時にサーバー時刻を記録する（クライアント時刻は参考値）

#### F-01-3: 休憩打刻

- 休憩開始・休憩終了の打刻（ICカード/Web両対応）
- 1日に複数回の休憩打刻が可能

#### F-01-4: 打刻修正申請

- 本人が修正理由と正しい時刻を入力して申請
- 承認者が承認すると打刻が修正される
- 修正履歴を保持する（誰が・いつ・何を修正したか）

### F-02: 勤務時間計算

#### F-02-1: 基本計算

- 所定労働時間: 7.5時間/日
- フレックスタイム制: コアタイム 10:00〜15:00
- 勤務時間 = 退勤時刻 − 出勤時刻 − 休憩時間
- 残業時間 = 勤務時間 − 所定労働時間（7.5h）（正の場合のみ）

#### F-02-2: 深夜勤務

- 22:00〜5:00 の勤務を自動判定
- 深夜勤務時間を別途集計する

#### F-02-3: 休日勤務

- 勤務カレンダーに基づき休日勤務を自動判定
- 休日勤務時間を別途集計する

### F-03: 月次勤怠一覧

- 自分の当月の日別勤怠一覧を表示
- 出勤時刻・退勤時刻・休憩時間・勤務時間・残業時間を表示
- 月の合計（勤務時間、残業時間、深夜時間、休日勤務時間）を表示
- 過去月の閲覧が可能

### F-04: 有給休暇管理

#### F-04-1: 有給付与（自動）

- 労働基準法に基づき入社日起算で自動付与
- 付与日数: 勤続年数に応じた法定日数
- 繰越: 前年度未消化分を翌年度に繰越（上限: 法定通り2年で時効）

#### F-04-2: 有給申請

- 全休 / 半休（午前半休・午後半休）の申請
- 申請 → 承認者が承認 → 残日数を自動減算
- 年5日取得義務の残日数を画面に表示

#### F-04-3: 年5日取得義務アラート

- 取得が進んでいない社員に対しアラートを表示
- 管理者画面で未取得者一覧を表示

### F-05: 残業管理

#### F-05-1: 残業自動計算

- 所定労働時間（7.5h）超過分を自動で残業として計算

#### F-05-2: 36協定アラート

- 月45時間超過時: 承認者・管理者にアラート通知
- 年360時間超過見込み時: 管理者にアラート通知
- 閾値超過時は承認者の承認が必要

#### F-05-3: 勤務間インターバル

- 前日退勤〜当日出勤が11時間未満の場合にアラート表示

### F-06: 承認者機能

- 自部門メンバーの勤怠一覧閲覧
- 打刻修正申請の承認・却下
- 有給申請の承認・却下
- 残業（月45h超過）の承認・却下
- 未処理の申請一覧表示

### F-07: 管理者機能

#### F-07-1: 社員マスタ管理

- 社員の登録・編集・無効化
- 所属部門の設定（兼務対応）
- ロールの割り当て

#### F-07-2: 部門マスタ管理

- 4階層（本部・部・課）の登録・編集・無効化
- 承認者の割り当て

#### F-07-3: 勤務カレンダー設定

- 祝日・会社休日の設定
- 年度単位で管理

#### F-07-4: 全社勤怠集計・レポート

- 全社員の月次勤怠集計
- 部門別の残業時間集計
- 36協定超過者一覧

#### F-07-5: アラート一覧

- 36協定超過・超過見込みアラート
- 有給5日未取得アラート
- 勤務間インターバル違反アラート

## 非機能要件

### 性能

- 想定ユーザー数: 1,000名
- 打刻のレスポンス: 2秒以内
- 一覧画面の表示: 3秒以内

### セキュリティ

- Microsoft Entra ID による SSO 認証
- ロールベースのアクセス制御
- 監査ログ（打刻修正履歴等）の保持

### 可用性

- 平日 7:00〜22:00 はサービス稼働（打刻可能）

### モバイル対応

- レスポンシブWebデザイン（ネイティブアプリ不要）
- スマートフォンからの打刻・勤怠確認が可能

## ユーザーストーリー

### 一般社員として

- 出勤時にICカードをかざして出勤打刻したい（退勤も同様）
- ICカードを忘れた日はWeb画面から打刻したい
- 打刻を忘れた場合に修正申請を出したい
- 自分の今月の勤怠一覧を確認したい
- 有給休暇を申請したい
- 有給の残日数を確認したい
- 残業時間が月の上限に近づいたら気づきたい

### 承認者として

- 部下の打刻修正申請を承認・却下したい
- 部下の有給申請を承認・却下したい
- 部下の残業が45時間を超える場合に承認判断したい
- 部下の勤怠状況を一覧で把握したい

### 管理者として

- 社員情報を登録・編集したい
- 部門構造を管理したい
- 勤務カレンダー（祝日・会社休日）を設定したい
- 全社の勤怠をレポートで確認したい
- 36協定違反リスクのある社員をアラートで把握したい
- 有給5日未取得の社員を把握したい

## 制約・前提

- ICカードリーダーとの通信プロトコルは別途 I/F 仕様書で定義する
- SSO 連携は Microsoft Entra ID の OIDC/OAuth2 を使用する
- 給与計算システムとの連携は本システムのスコープ外（CSV出力で対応）
- 勤務カレンダーの祝日データは管理者が手動設定する
